# API8:2023 — Security Misconfiguration

API-specific configuration gaps. See also [vuln-checklists/A05](../vuln-checklists/A05-security-misconfiguration.md).

## API-specific surfaces

- CORS: reflective `Access-Control-Allow-Origin` with `Allow-Credentials`.
- Verbose errors: stack traces, framework versions, SQL fragments leak in API responses.
- Default CRUD endpoints generated by frameworks left enabled.
- Swagger / OpenAPI / GraphQL introspection exposed in production.
- Authentication header echoing back in error responses (`"Invalid token: <token>"`).
- Outdated API versions still routable.

## Hardening

- Per-environment configs. Production has different defaults than dev.
- CI check: refuse builds that leave debug routes wired.
- Strict CORS allowlists. No reflective behaviors.
